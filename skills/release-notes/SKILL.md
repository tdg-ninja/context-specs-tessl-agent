# release-notes

Follow this compact workflow. Read `references/full-instructions.md` before drafting release notes.

1. Identify the requested release, PR set, git range, tags, or last-release-to-HEAD source.
2. Load the referenced instructions and collect the source evidence.
3. Draft concise release notes with highlights, changes, fixes, internal / harness changes, and upgrade notes.
4. Cite PR numbers or commit refs when available.
5. End with any follow-up questions needed before publishing the notes.
